# Бэкенд-платформа для площадки перепродажи вещей (Аналог Авито)

Данный проект представляет собой выпускную дипломную работу по программе обучения «Java-разработчик». Приложение реализует полноценную бэкенд-часть для платформы C2C-торговли, обеспечивая управление объявлениями, комментариями, профилями пользователей и медиафайлами.

## Используемый технологический стек
* **Java 11** — базовая платформа разработки.
* **Spring Boot 2.7.15** — фреймворк для быстрого развертывания приложения.
* **Spring Security** — обеспечение безопасности и разграничения прав с использованием Basic Authentication.
* **Spring Data JPA** — объектно-реляционное отображение и управление доступом к данным.
* **PostgreSQL** — реляционная СУБД для хранения информации в production-режиме.
* **H2 Database** — встроенная БД для изолированного прогона интеграционных тестов.
* **Liquibase** — система управления и версионирования миграций базы данных.
* **MapStruct 1.5.5.Final** — высокопроизводительный маппер для конвертации Java-бинов (Entity <-> DTO).
* **Lombok** — библиотека для автоматической генерации шаблонного кода (геттеры, сеттеры, конструкторы).
* **Springdoc-openapi (Swagger UI) 1.7.0** — автоматическая генерация интерактивной документации REST API.

## Архитектура и структура проекта
Проект строго следует принципам слоистой архитектуры (Layered Architecture) и принципу единственной ответственности (SRP):
1. **Controller Layer (`ru.skypro.homework.controller`)**: Принимает HTTP-запросы от фронтенда, выполняет первичную валидацию данных и описывает контракты API для Swagger.
2. **Service Layer (`ru.skypro.homework.service`)**: Содержит всю бизнес-логику приложения, управляет транзакциями и проверяет права доступа на уровне методов с помощью аннотаций `@PreAuthorize`.
3. **Security Layer (`ru.skypro.homework.security`)**: Кастомная интеграция со Spring Security через `UserDetailsService` и `UserDetails` для аутентификации пользователей по данным из PostgreSQL.
4. **Repository Layer (`ru.skypro.homework.repository`)**: Интерфейсы, расширяющие `JpaRepository`, для прямого взаимодействия с таблицами БД.
5. **Mapper Layer (`ru.skypro.homework.mapper`)**: Интерфейсы MapStruct для эффективного и безопасного преобразования сущностей в объекты передачи данных (DTO) и обратно.
6. **Model Layer (`ru.skypro.homework.model`)**: JPA-сущности, описывающие структуру таблиц в базе данных. All `@ManyToOne` связи используют оптимизацию `FetchType.LAZY`.

## Модель безопасности и роли пользователей
Доступ к эндпоинтам разграничен на основе ролей (`USER`, `ADMIN`):
* **Анонимные пользователи**: Могут просматривать список всех объявлений (`GET /ads`) и запрашивать загруженные медиафайлы.
* **Авторизованные пользователи (`USER`)**: Имеют полный доступ к просмотру контента, могут создавать объявления, оставлять комментарии, а также редактировать и удалять **только свои** собственные объявления и комментарии.
* **Администраторы (`ADMIN`)**: Имеют наивысшие права и могут редактировать или удалять любые объявления и комментарии в системе.

## Схема базы данных (Миграции)
Управление базами данных полностью автоматизировано через Liquibase. Настроены каскадные правила (`ON DELETE CASCADE`), гарантирующие целостность данных:
* При удалении пользователя автоматически удаляются его объявления и комментарии.
* При удалении объявления автоматически удаляются связанные с ним комментарии.
* Бинарные данные изображений и аватарок надежно хранятся в формате `BYTEA` (PostgreSQL) для изоляции медиаконтента в рамках СУБД.

## Инструкция по локальному запуску приложения
1. **Запуск фронтенд-части**: Убедитесь, что у вас запущен Docker Desktop. Откройте терминал и выполните команду:
   ```bash
   docker run -p 3000:3000 --rm ghcr.io/dmitry-bizin/front-react-avito:v1.21
   ```
   После этого фронтенд станет доступен в браузере по адресу: `http://localhost:3000`.
2. **Подготовка базы данных**: Создайте в вашей локальной СУБД PostgreSQL пустую базу данных с именем `avito`. Параметры подключения по умолчанию (`postgres`/`postgres`) уже прописаны в `application.properties`.
3. **Запуск бэкенда**: Запустите проект из вашей IDE через главный класс `HomeworkApplication`. Liquibase автоматически применит все чейнджсеты и создаст таблицы.
4. **Проверка API**: Документация Swagger UI и интерактивный интерфейс тестирования эндпоинтов доступны по адресу: `http://localhost:8080/swagger-ui/index.html`.
